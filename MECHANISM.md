# 오큐몽 — 피로도 산출 및 개입 메커니즘

## 전체 흐름

```
카메라 입력 (30fps)
    ↓
MediaPipe FaceMesh → 468개 랜드마크 추출
    ↓
EAR 계산 (눈 개폐 비율)
    ↓
베이스라인 비교 → 현재 피로도 / 컨디션 지수 산출
    ↓
화면 개입 (dim overlay) + 알림
```

---

## 1. 베이스라인 수집

측정 시작 직후 약 **30초**간 사용자의 평상시 눈 상태를 수집한다.

| 수집 항목 | 설명 |
|---|---|
| `baselineRef.ear` | 눈을 뜬 상태의 평균 EAR |
| `baselineRef.blinkRate` | 분당 깜빡임 횟수 (blinks/min) |
| `baselineRef.blinkDur` | 깜빡임 1회 평균 지속 시간 (ms) |

수집 완료 후 **2.5분(150초)** 간 워밍업 단계가 이어지며, 이 기간에도 베이스라인 값이 지속 업데이트되어 기준값이 안정화된다.

> **하루 단위 재사용**: 당일 세션 파일에 `baselineRef`가 저장되어 있으면 앱 재시작 시 베이스라인 수집을 건너뛰고 바로 측정을 시작한다.

---

## 2. EAR (Eye Aspect Ratio)

눈의 개폐 정도를 나타내는 비율. 6개의 눈 랜드마크 좌표로 계산한다.

```
EAR = (|p2−p6| + |p3−p5|) / (2 × |p1−p4|)
```

- 눈을 완전히 뜬 상태: ~0.30 내외
- 눈이 감기면 → 0에 가까워짐
- 좌우 눈의 평균값을 사용

깜빡임 감지: EAR이 임계값(동적 결정) 이하로 떨어졌다가 올라오면 1회 깜빡임으로 기록한다.

---

## 3. 현재 피로도 산출 (`computeFatigue`)

**최근 30초 롤링 윈도우** 기준으로 4가지 지표를 가중 합산한다.

### 지표별 계산

| 지표 | 가중치 | 계산 방법 |
|---|---|---|
| EAR 하락도 | **35%** | `(1 − avgEAR / baselineEAR) × 5` |
| 깜빡임 빈도 편차 | **20%** | `|현재 빈도 / 베이스라인 빈도 − 1| × 1.5` |
| 깜빡임 지속시간 | **20%** | `(avgDur / baselineDur − 1) × 2` |
| PERCLOS | **25%** | `눈 감긴 시간 비율 × 8` |

```
현재 피로도 = (earScore×0.35 + blinkRateScore×0.20 + blinkDurScore×0.20 + perclosScore×0.25) × 100
```

- 범위: **0~100**, 높을수록 지금 눈이 피로한 상태
- 2초마다 재계산, 상세측정 탭에 표시
- 각 지표 점수는 0~1로 클램핑

### PERCLOS
30초 윈도우 내 전체 프레임 중 눈이 감긴 프레임의 비율.  
PERCLOS > 10%는 강한 졸음/피로 신호로 간주된다.

## 4. 컨디션 지수 산출

**100점에서 시작**, 1분마다 현재 피로도에 비례하여 감점한다. 자연 회복은 없으며 **20-20-20 회복 루틴 완료 시 +10점**만 부여된다.

### 감점 공식 (1분마다 갱신)

```
delta = -(현재 피로도 / 100) × 5 × (sensitivity / 5)
```

- 현재 피로도가 높을수록 감점 폭이 커짐
- `sensitivity`는 설정 슬라이더 1~10 값, 기본값 5일 때 배율 1.0
- 현재 피로도가 0이면 감점 없음

### 감점 예시 (sensitivity = 5 기준)

| 현재 피로도 | 분당 감점 | 100→60점 도달 |
|---|---|---|
| 20점 | −1.0점/분 | 약 40분 |
| 40점 | −2.0점/분 | 약 20분 |
| 60점 | −3.0점/분 | 약 13분 |
| 80점 | −4.0점/분 | 약 10분 |
| 100점 | −5.0점/분 | 약 8분 |

### 회복
- 20-20-20 루틴(20초 눈 운동) 완료 시 `+10점` 부여
- 세션 시작 시 항상 100점에서 출발

---

## 5. 개입 시스템

컨디션 지수에 따라 화면 밝기를 자동으로 낮춘다. **전체화면 투명 오버레이(BrowserWindow)** 를 화면 위에 띄우는 방식으로 구현되어 있다.

### 개입 단계

| 컨디션 지수 | 오버레이 불투명도 | 상태 |
|---|---|---|
| 100 ~ 70 | 0% (없음) | 정상 |
| 70 ~ 50 | 10% | 경고 |
| 50 ~ 35 | 20% | 주의 |
| 35 이하 | 30% | 위험 |

- 오버레이는 `alwaysOnTop: 'screen-saver'` 레벨로 모든 창 위에 표시된다
- 불투명도 변화는 step 0.02씩 페이드 인/아웃 처리 (50ms 간격)
- **집중 모드** 활성화 시 개입 전체 중단

---

## 6. 알림 시스템

`node-notifier`를 통해 macOS 네이티브 알림을 발송한다.

| 트리거 조건 | 알림 내용 |
|---|---|
| 컨디션 지수 60점 하향 돌파 | "깜빡임이 줄어들고 있어요. 잠깐 눈을 쉬어보세요." |
| 컨디션 지수 35점 하향 돌파 | "지금 바로 20-20-20 루틴을 시작해보세요." (클릭 시 운동 탭 이동) |

- **설정 > 데스크톱 알림** 토글로 비활성화 가능
- **집중 모드** 활성화 시 알림 발송 중단
- 동일 임계값은 한 번 돌파 후 점수가 회복되어 다시 돌파할 때만 재발송

## 7. 데이터 저장

### 세션 파일 위치
```
~/Library/Application Support/okumong/sessions/YYYY-MM-DD.json
```

### 저장 시점
- 10분마다 자동 저장
- 앱 정상 종료 시 최종 저장 (`before-quit` → `await saveSession()` → `quit-ready`)

### 저장 항목

```json
{
  "date": "YYYY-MM-DD",
  "summary": {
    "avgFatigue": 32,
    "peakFatigue": 78,
    "warnCount": 2,
    "dangerCount": 1,
    "recoveryCount": 1,
    "measureDurationMin": 145

  "baselineRef": { "ear": 0.312, "blinkRate": 18.4, "blinkDur": 142 },
  "events": [ { "type": "", "msg": "...", "ts": 1747000000000 } ],
  "dataLog": [
    {
      "timestamp": "2026-05-12T...",
      "elapsed_sec": 600,
      "ear_left": 0.31,
      "ear_right": 0.30,
      "ear_avg": 0.305,
      "ear_baseline": 0.312,
      "ear_ratio": 0.977,

```

`dataLog`는 10분 간격으로 기록되며 EAR·피로도 분석에 활용할 수 있다.
